Iterating Through Lists Part I

When you render a list of items, React always stores some info about each item in the list. If you render a component that has state, that state needs to be stored and regardless of how you implement your components, React stores a reference to the backing natives views. When you update that list, React need to determine what has changed.

You could've added, remved, rearranged, or updated items in the list. "Key" is a special property that's reserved by React. When an element is created, React pulls off the key property and stores the key directly on the returned element. Even though it may look like it is part of props, it cannot be referenced with "this.props.key".
