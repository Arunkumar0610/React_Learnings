Class Components: Javascript classes introduced in ECMAScript 2015(ES6) are primarily syntactical sugar over Javascript's existing prototype-based inheritance The class syntax is not introducing a new object-oriented inheritance model to JavaScript. JavaScript classes provide a much simpler and clearer syntax to create objects and deal with inheritance.

If using Functional Stateless Components is Best Practice Then why Class Components?
Rule1: If we want to use this keyword we need to use Class Components.
Rule2: If we want to use the Lifecycle Methods then we need to use Class Components.
